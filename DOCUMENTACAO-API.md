📦 api-ofertas — Documentação TécnicaBackend do sistema Admin Ofertas — API REST para gestão de ofertas de supermercado1. 📋 Visão GeralA api-ofertas é uma API REST robusta desenvolvida para centralizar a gestão de campanhas promocionais em redes de varejo. Construída sobre a stack Fastify, Prisma e PostgreSQL, a aplicação utiliza uma arquitetura multi-tenant isolada por domínio, garantindo que múltiplos clientes operem de forma independente no mesmo ambiente. O sistema é otimizado para alta performance em operações de leitura e processamento de dados em lote através de importações via arquivos CSV.2. 🛠️ Stack TecnológicaTecnologia / BibliotecaVersão / FunçãoNode.js20 Alpine (Runtime)Fastify^5.8.5 (Framework HTTP)Prisma^6.19.3 (ORM)PostgreSQLBanco de Dados Relacional@fastify/jwt^10.0.0 (Autenticação)csv-parse^6.2.1 (Processamento de Arquivos)bcrypt^6.0.0 (Criptografia de Senhas)@fastify/cors^11.2.0 (Segurança de Origem)@fastify/multipart^10.0.0 (Upload de Arquivos)3. 📁 Estrutura de Diretóriosapi-ofertas/

4. 🗄️ Modelo de DadosO banco de dados é estruturado para suportar a complexidade de múltiplas operações simultâneas. Os principais modelos incluem:tenants: Entidade raiz que define o cliente, contendo nome, slug único e o domínio associado para identificação da requisição.users: Usuários administrativos vinculados a um tenant, com armazenamento seguro de hash de senha e definição de papéis (roles).stores: Cadastro das filiais físicas, incluindo dados de localização (cidade, estado, endereço) e contato telefônico.categories: Organização lógica do catálogo de produtos para facilitar a navegação e filtragem.products: Cadastro técnico de itens com código interno, código de barras (EAN), descrição e unidade de medida.offers: Regras de negócio das promoções, vinculando produtos a lojas com preços de/por e período de validade.csv_imports: Registro de auditoria para importações, armazenando status, total de linhas processadas e logs de erro.media_assets: Gerenciamento de referências para arquivos de mídia e imagens dos produtos cadastrados.5. 🔐 Autenticação e SegurançaA segurança é implementada através de JSON Web Tokens (JWT). O sistema utiliza decorators personalizados como authenticate para validar a sessão do usuário e authorize('admin') para restringir operações críticas. A identificação do tenant ocorre de forma transparente através do header Host da requisição, garantindo que os dados retornados pertençam exclusivamente ao domínio solicitante.6. 🐳 Docker e ContainerizaçãoA aplicação é totalmente containerizada utilizando uma imagem base node:20-alpine para garantir leveza e segurança. O Dockerfile realiza o build das dependências e a geração do Prisma Client em tempo de montagem. O arquivo docker-compose.yml orquestra o serviço da API juntamente com a instância do banco de dados PostgreSQL, facilitando a replicação do ambiente de desenvolvimento.7. 🔄 Fluxo de CI/CDO ciclo de entrega contínua é automatizado via GitHub Actions. Ao realizar um push na branch main, o pipeline inicia o build da imagem Docker, realiza o push para o GitHub Container Registry (ghcr.io) e dispara um webhook para o Portainer, que executa o redeploy automático dos serviços em produção.8. 🌐 Infraestrutura e DeployO ambiente de produção utiliza Docker Swarm para orquestração de containers. O tráfego é gerenciado pelo Traefik, que atua como reverse proxy e gerencia a emissão de certificados SSL via Let's Encrypt. A gestão da infraestrutura é realizada através do Portainer Enterprise Edition, permitindo monitoramento em tempo real e controle de versões.9. 📡 Endpoints da APIMóduloMétodoRotaDescriçãoAuthPOST/auth/loginAutenticação de usuário e geração de tokenProductsGET/productsListagem de produtos com busca e paginaçãoProductsGET/products/:idDetalhes de um produto específicoProductsPOST/productsCadastro de novo produtoProductsPUT/products/:idAtualização de dados do produtoProductsDELETE/products/:idRemoção de produto do catálogoStoresGET/storesListagem de todas as filiaisStoresGET/stores/:idDetalhes de uma filialStoresPOST/storesCadastro de nova filialStoresPUT/stores/:idAtualização de dados da filialStoresDELETE/stores/:idRemoção de filialCategoriesGET/categoriesListagem de categorias cadastradasCategoriesGET/categories/:idDetalhes de uma categoriaCategoriesPOST/categoriesCadastro de nova categoriaCategoriesPUT/categories/:idAtualização de categoriaCategoriesDELETE/categories/:idRemoção de categoriaOffersGET/offersListagem de ofertas ativasOffersGET/offers/:idDetalhes de uma ofertaOffersPOST/offersCriação de nova ofertaOffersPUT/offers/:idEdição de oferta existenteOffersDELETE/offers/:idExclusão de ofertaImportsPOST/imports/csvImportação de ofertas via CSVImportsPOST/imports/storesImportação de filiais via CSVImportsPOST/imports/categoriesImportação de categorias via CSVSystemGET/Health check do sistemaDocumentação gerada em 17 de maio de 2026.
